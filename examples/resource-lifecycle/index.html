What this proves:
  • autoLoad: true resource reaches ready on runtime start.
  • autoLoad: false resource starts idle.
  • resource.reload() transitions through pending to ready.
  • resource.invalidate() marks stale without reloading.
  • reload() clears stale.
  • resource.load() manually triggers a deferred load.
  • action.invalidates(resource) marks the resource stale after successful action.
  • A failed loader produces failed status and error.
  • A failed action does not invalidate its targeted resources.
  • Route context is available to resource loaders.
